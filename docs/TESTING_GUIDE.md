# 工具页面功能测试指南

## 测试前准备

1. 确保已安装依赖：`npm install`
2. 确保 `skills` 目录存在并包含示例 Skills
3. 启动开发服务器：`npm run dev`

## 测试步骤

### 1. 页面加载测试

- [ ] 打开工具页面
- [ ] 验证所有工具卡片正确显示
- [ ] 验证工具图标正确加载
- [ ] 验证工具状态标签显示正确（已检测到/未检测到）

### 2. 标签筛选测试

- [ ] 点击"全部"标签，验证显示所有工具
- [ ] 点击"已安装"标签，验证只显示已检测到的工具
- [ ] 点击"未安装"标签，验证只显示未检测到的工具

### 3. 工具开关测试

#### 3.1 首次开启工具

- [ ] 选择一个未开启的工具
- [ ] 点击开关按钮
- [ ] 验证弹出 Skills 列表弹窗
- [ ] 验证弹窗显示所有可用的 Skills
- [ ] 选择几个 Skills 并启用
- [ ] 关闭弹窗
- [ ] 验证工具开关状态变为开启
- [ ] 检查工具的 skills 目录，验证选中的 Skills 已被拷贝

#### 3.2 非首次开启工具

- [ ] 关闭刚才开启的工具
- [ ] 再次开启该工具
- [ ] 验证不弹出 Skills 列表（直接根据配置拷贝）
- [ ] 验证之前启用的 Skills 被自动拷贝

#### 3.3 关闭工具

- [ ] 点击开关关闭工具
- [ ] 验证工具开关状态变为关闭
- [ ] 检查工具的 skills 目录，验证所有 Skills 已被删除

### 4. Skills 管理测试

- [ ] 开启一个工具
- [ ] 点击设置按钮（⚙图标）
- [ ] 验证弹出 Skills 列表弹窗
- [ ] 验证显示工具图标和名称
- [ ] 验证显示所有可用的 Skills
- [ ] 验证每个 Skill 的启用状态正确显示

#### 4.1 启用 Skill

- [ ] 点击一个未启用的 Skill 的"启用"按钮
- [ ] 验证按钮文字变为"禁用"
- [ ] 验证 Skill 状态变为"已启用"
- [ ] 验证 Skill 卡片样式变化（高亮）
- [ ] 检查工具的 skills 目录，验证 Skill 已被拷贝

#### 4.2 禁用 Skill

- [ ] 点击一个已启用的 Skill 的"禁用"按钮
- [ ] 验证按钮文字变为"启用"
- [ ] 验证 Skill 状态变为"未启用"
- [ ] 验证 Skill 卡片样式恢复
- [ ] 检查工具的 skills 目录，验证 Skill 已被删除

#### 4.3 统计信息

- [ ] 验证弹窗底部显示正确的 Skills 总数
- [ ] 验证显示正确的已启用 Skills 数量
- [ ] 启用/禁用 Skills 后，验证统计数字实时更新

### 5. 路径复制测试

- [ ] 点击配置路径旁边的复制按钮
- [ ] 验证路径已复制到剪贴板
- [ ] 点击 Skills 路径旁边的复制按钮
- [ ] 验证路径已复制到剪贴板

### 6. 刷新功能测试

- [ ] 点击刷新按钮
- [ ] 验证工具列表重新加载
- [ ] 验证工具状态更新正确

### 7. 配置持久化测试

- [ ] 启用几个工具和 Skills
- [ ] 关闭程序
- [ ] 重新启动程序
- [ ] 验证工具开关状态保持
- [ ] 打开 Skills 管理弹窗
- [ ] 验证 Skills 启用状态保持

### 8. 错误处理测试

- [ ] 删除 `skills` 目录
- [ ] 刷新工具列表
- [ ] 验证程序不崩溃，显示空 Skills 列表
- [ ] 恢复 `skills` 目录
- [ ] 验证功能恢复正常

## 测试数据位置

- **Skills 模板目录**：`skills-manager-app/skills/`
- **工具配置目录**：`skills-manager-app/data/`（自动创建）
- **工具 Skills 目录**：`~/.{工具名}/skills/`（如 `~/.kiro/skills/`）

## 预期结果

所有测试项都应该通过，程序应该：

1. 正确显示所有工具和状态
2. 正确处理工具开关操作
3. 正确管理 Skills 的启用/禁用
4. 正确拷贝和删除 Skills 目录
5. 正确保存和加载配置
6. 界面响应流畅，无卡顿
7. 错误处理得当，不崩溃
